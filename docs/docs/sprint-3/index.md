# Sprint 3

 A Sprint 3 marcou a consolidação da integração entre backend, frontend web, interface TUI e pipeline de visão computacional do projeto. Durante esta etapa, os esforços foram concentrados na evolução da arquitetura do sistema, integração entre serviços e aprimoramento do reconhecimento automático de placas veiculares (ALPR).

 A sprint também contemplou testes e validações de diferentes modelos YOLO, integração do OCR ao pipeline e melhorias na experiência operacional do sistema.

---

## Objetivos

* Consolidar a integração entre backend e frontend;
* Validar o fluxo de aprovação e rejeição de scans;
* Integrar consumo de dados reais da API;
* Evoluir a interface operacional utilizando TUI;
* Aprimorar o pipeline de visão computacional;
* Investigar melhorias de desempenho em inferência em tempo real;
* Estruturar melhor a documentação técnica do projeto.

---

## Entregas

* Integração do frontend com backend e autenticação;
* Consumo de dados reais da API nas principais telas;
* Implementação inicial do fluxo de validação de scans;
* Implementação da interface TUI;
* Refinamento e testes comparativos entre modelos YOLO;
* Integração do OCR ao pipeline de reconhecimento;
* Validação do reconhecimento de placas com o drone DJI Tello;
* Documentação técnica das integrações e arquitetura do sistema.

---

## Artefatos

### Implementação Técnica

 Este artefato contempla o desenvolvimento técnico dos componentes do sistema, incluindo frontend, backend, TUI, integração entre serviços e evolução do pipeline de visão computacional.

### Documentação

 Este artefato reúne a documentação produzida durante a sprint, incluindo arquitetura do sistema, integração entre serviços, backend, banco de dados e evolução do pipeline de visão computacional.
