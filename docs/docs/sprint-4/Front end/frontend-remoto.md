# Frontend Remoto

Durante a Sprint 4, foram realizadas diversas melhorias no Frontend Remoto para otimizar a experiência do usuário e facilitar a gestão das operações e dos alertas. O design foi refinado para tornar o uso mais fluido e a visualização dos dados mais clara.

Abaixo estão os locais e os prints das telas e componentes desenvolvidos, comprovando as melhorias feitas.

## Tela Principal do Gestor Remoto

A tela principal do gestor remoto apresenta os veículos que foram identificados pelo sistema como roubados. Esses registros ficam aguardando a validação manual do usuário para verificar se o OCR (reconhecimento óptico de caracteres) leu corretamente os caracteres da placa.

![Tela Principal Gestor Remoto](/img/sprint-4/frontend/telaprincipalgestorremoto.png)

### Card da Tela Principal

Os cards presentes na tela principal foram ajustados para exibir os dados de forma mais acessível e direta, facilitando a validação manual.

![Card Tela Principal Gestor Remoto](/img/sprint-4/frontend/cardtelaprincipalgestorremoto.png)

## Veículos Confirmados

Esta tela exibe a listagem dos veículos que foram validados positivamente pelo gestor remoto, confirmando a correspondência com os dados do veículo roubado.

![Tela Confirmados Gestor Remoto](/img/sprint-4/frontend/telaconfirmadosgestorremoto.png)

### Card de Veículos Confirmados

Exibe as informações consolidadas do veículo suspeito validado pelo gestor.

![Card Confirmados Gestor Remoto](/img/sprint-4/frontend/cardconfirmadosgestorremoto.png)

## Veículos Recusados

Esta tela exibe os registros que foram recusados pelo gestor remoto durante a etapa de validação (por exemplo, quando a leitura do OCR estava incorreta ou o carro não era o procurado).

![Tela Recusados Gestor Remoto](/img/sprint-4/frontend/telarecusadosgestorremoto.png)

### Card de Veículos Recusados

Exibe as informações do veículo e a justificativa da recusa feita pelo gestor.

![Card Recusados Gestor Remoto](/img/sprint-4/frontend/cardrecusadosgestorremoto.png)
